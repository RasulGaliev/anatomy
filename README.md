# Anatomia — анатомия: русский ↔ латынь

Учебное веб-приложение для студентов-медиков: изучение анатомических терминов
с переводом между русским и латынью. **725 терминов**, разбитых по системам организма.

Карточки, тесты и поиск — всё работает в браузере, без сервера и установки.

## Возможности

- **Карточки** — переворот по клику, перемешивание, переход вперёд/назад.
- **Тест** — 10 вопросов с 4 вариантами, подсветка ответов, результат в процентах.
- **Список** — поиск по русскому и латыни, фильтр по разделу.
- **Направление** перевода RU → LA и LA → RU.
- **9 разделов**: кости, суставы, мышцы, пищеварительная, дыхательная,
  мочеполовая системы, сердце, артерии, вены.

## Запуск локально

Просто откройте `index.html` двойным кликом — приложение работает напрямую
из файловой системы (данные подключаются через `assets/js/words.js`, а не через `fetch`,
поэтому локальный сервер не нужен).

## Публикация на GitHub Pages

1. Создайте репозиторий и запушьте содержимое этой папки.
2. **Settings → Pages → Build and deployment**: источник `Deploy from a branch`,
   ветка `main`, папка `/ (root)`.
3. Сайт будет доступен по адресу `https://<логин>.github.io/<репозиторий>/`.

Все пути к ресурсам относительные, а файл `.nojekyll` отключает обработку Jekyll —
поэтому сайт работает в подпапке `github.io` без дополнительной настройки.

## Формат данных

`data/words.json`:

```json
{ "id": 1, "ru": "Шейный позвонок", "la": "vertebra cervicalis", "cat": "bones" }
```

| Поле | Описание                                   |
|------|--------------------------------------------|
| `id` | номер термина (1–725)                       |
| `ru` | термин на русском                           |
| `la` | термин на латыни                            |
| `cat`| ключ раздела (`bones`, `muscles`, `heart`…) |

## Обновление списка терминов

Источник правды — `src/doc` (строки вида `1. Русский термин – latin term`).
После правки перегенерируйте данные из корня проекта:

```bash
node src/parse.js
```

Скрипт пересоберёт `data/words.json` и `assets/js/words.js`. Требуется Node.js.
